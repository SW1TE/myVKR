## 📋 Описание

Проект представляет собой программную реализацию **отказоустойчивой событийно-ориентированной архитектуры** на базе Apache Kafka для распределённых информационных систем. Разработан в рамках выпускной квалификационной работы в НИЯУ МИФИ.

### 🔑 Ключевые возможности

- **Гарантия exactly-once** — сочетание идемпотентного продюсера, транзакционного Outbox и дедупликации на стороне потребителя
- **Отказоустойчивость** — репликация данных (RF=3), автоматическое переключение лидера, восстановление после сбоев
- **Dead Letter Queue** — изоляция проблемных сообщений без блокировки основной очереди
- **Мониторинг** — интеграция с Prometheus, Grafana и OpenTelemetry
- **Соответствие стандартам** — ГОСТ Р ИСО/МЭК 25010, ГОСТ Р ИСО/МЭК 27001

---


### Путь сообщения

1. **Producer Service** получает REST-запрос → сохраняет бизнес-данные и событие в Outbox в одной ACID-транзакции
2. **Outbox Publisher** (фоновый процесс) периодически отправляет неотправленные сообщения в Kafka с `acks=all` и `enable.idempotence=true`
3. **Kafka Cluster** реплицирует сообщение на ≥2 брокера (min.insync.replicas=2)
4. **Consumer Service** читает сообщение → проверяет UUID в `processed_events` → выполняет бизнес-логику → сохраняет факт обработки
5. При ошибке: **транзиентная** → повтор с экспоненциальным backoff; **перманентная** → перенос в Dead Letter Queue с фиксацией смещения
